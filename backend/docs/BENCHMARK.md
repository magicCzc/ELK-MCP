# 性能基准报告模板

## 目标

- 查询响应时间 P99 < 500ms
- 支撑 100+ 并发查询
- 冷启动 ≤ 2s

## 环境说明

- 硬件/CPU/内存/网络
- ES 集群规模与分片配置
- 测试数据规模（索引大小、文档数）

## 方法论

- 使用固定查询模板与变更过滤条件进行压测
- 命中缓存与未命中缓存分别统计
- 记录 ES 后端响应时间与应用层处理时间

## 指标与结果

- 延迟：P50/P90/P99（ms）
- 吞吐：RPS
- 资源：CPU/内存/连接数

## 优化建议与结论

- 提升缓存命中率与 TTL 调整
- 优化查询 DSL 与索引字段映射
- 连接池与并发策略调整
